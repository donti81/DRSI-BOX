# Download Tracking — Implementacijski log

**Datum:** 2026-05-08  
**Cilj:** Beleženje prenosov datotek na FileManager strani z Oracle bazo.

---

## Kontekst

Prenosi datotek so bili direktne statične povezave (`/uploads/{guid}.ext`) brez sledenja. "Downloads" gumb v meniju ni delal ničesar. Implementacija doda Oracle tabelo za beleženje prenosov, nov download endpoint, in Downloads pogled.

---

## Arhitektura

```
Uporabnik klikne "Download"
    → GET /apps/file-manager?handler=Download&id={id}
    → FileManager.cshtml.cs :: OnGetDownloadAsync(long id)
        → UploadLogService.GetByIdAsync(id)         — poišče pot datoteke
        → DownloadLogService.LogAsync(id, ip)        — zabeleži prenos v Oracle
        → PhysicalFile(path, contentType, name)      — streama datoteko z Content-Disposition: attachment
```

---

## Oracle tabela

```sql
-- Scripts/CreateDownloadLogs.sql
CREATE TABLE download_logs (
    id              NUMBER          GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    upload_log_id   NUMBER          NOT NULL,
    downloaded_at   TIMESTAMP       DEFAULT CURRENT_TIMESTAMP NOT NULL,
    ip_address      VARCHAR2(50),
    CONSTRAINT fk_download_upload FOREIGN KEY (upload_log_id)
        REFERENCES upload_logs (id) ON DELETE CASCADE
);
CREATE INDEX ix_download_logs_upload_id ON download_logs (upload_log_id);
```

`ON DELETE CASCADE` — ob brisanju datoteke iz `upload_logs` se download_logs samodejno zbriše.

## Servisne metode

### IDownloadLogService
| Metoda | SQL | Namen |
|--------|-----|-------|
| `LogAsync(uploadLogId, ipAddress, downloadedBy)` | `INSERT INTO download_logs ...` | Zabeleži en prenos |
| `GetCountsAsync()` | `SELECT upload_log_id, COUNT(*) ... GROUP BY` | Vrne `Dictionary<long, int>` za My Files pogled |
| `GetDownloadedFilesWithCountsAsync()` | `SELECT ... INNER JOIN download_logs ... GROUP BY` | Vrne datoteke z štetji za Downloads pogled |

### IUploadLogService (dodano)
| Metoda | SQL | Namen |
|--------|-----|-------|
| `GetByIdAsync(id)` | `SELECT ... WHERE id = :id` | Poišče zapis pred streamanjem datoteke |

## Spremembe na strani (FileManager.cshtml)

- **Sidebar "My Files"**: aktivna klasa glede na `Model.View != "downloads"`
- **Sidebar "Downloads"**: link → `/apps/file-manager?view=downloads`, prikaže badge s številom prenesenih datotek
- **Vse download povezave**: zamenjane iz `/uploads/{file}` v `/apps/file-manager?handler=Download&id={id}`
- **My Files tabela**: stolpec "Uploaded by" (prej "Owner") + stolpec "Downloads"
- **Downloads tabela**: stolpca "Downloaded by" (zadnji) in "Uploaded by"
- **Pogojni pogled**: `@if (Model.View == "downloads")` prikaže Downloads tabelo, sicer My Files

## Verifikacija

1. Zaženi `Scripts/CreateDownloadLogs.sql` na `localhost:1521/XEPDB1` kot `ITROHA`
2. Zaženi `Scripts/AlterTablesAddUser.sql` na `localhost:1521/XEPDB1` kot `ITROHA`
3. Restart serverja (`dotnet run`)
4. Naloži datoteko — v `upload_logs.uploaded_by` se zapiše email
5. Klikni Download — datoteka se prenese, v `download_logs.downloaded_by` se zapiše email
6. My Files tabela — stolpec "Uploaded by" prikazuje email nalagalca
7. Downloads tabela — stolpca "Downloaded by" in "Uploaded by" prikazujeta emaile
8. Zbriši datoteko — download_logs zapisi se samodejno zbrišejo (CASCADE)
